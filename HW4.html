Problem 2: density¶

Now you can make a scatter plot of just a 100,000 points, so you might try it to see what it looks like. But particularly in the central areas the points all overlap into just a blob, and this just gets worse as the number of points goes up. What is better is to plot the density of the points. There are several ways of doing this, but a 2D histogram plot is the easiest (plt.hist2d() in python and histogram2() in MatLab).

2a) Make a 2d histogram plot. Read about options, and play with binsize so it is useful (too large a bin and everything washes out, too small and the sample noise dominates). Make sure you use a colorbar.

I chose a bin size of 200 so that each period of the data correspond to a width of 4 bins.

2b) Clearly explain what is being plotted in your plot (e.g. what does each color mean?)

In [11]:

plt.ylabel("Background", fontsize = 24)
plt.title("2D Histogram of All 100,000 Data Points", fontsize = 24)
plt.show()

The graph above plots a 2D histogram of the whole data set with 200*200 bins. The approximate number of data points included in each bin can be told by comparing the bin's color to the color bar. For example, by looking at the color bar, we can tell that green, yellow, and red bins correspond to approximately 0-8, 8-16, and 16-24 counts, respectively.

# Calculate means
n = 20
t2 = np.linspace(0, t_width, n+1)
means = np.zeros(n)
for i in range(0,n):
    points = bkgd[(t_folded >= t2[i]) & (t_folded < t2[i+1])]
    means[i] = np.sum(points) / len(points)
t2 = t2[0:n] + (t2[1]-t2[0])/2

# Make plot
